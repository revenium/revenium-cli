    - TestRenderTable_Quiet: RenderTable with quiet Formatter produces no output
    - TestRenderTable_HasHeaders: Output contains all header strings
    - TestRenderTable_HasData: Output contains all data cell values
    - TestStatusStyle: statusStyle("active") returns green, statusStyle("inactive") returns red, statusStyle("pending") returns yellow
  </behavior>
  <action>
    Create `internal/output/` package with three production files:

    **output.go** — Formatter type:
    - `type Formatter struct` with fields: writer (io.Writer), errWriter (io.Writer), jsonMode (bool), quiet (bool), width (int), isTTY (bool)
    - `func New(jsonMode, quiet bool) *Formatter` — detects TTY via `term.IsTerminal(os.Stdout.Fd())`, gets terminal width via `term.GetSize(os.Stdout.Fd())` (default 80 on failure), sets writer to `io.Discard` if quiet && !jsonMode
    - Expose `IsJSON() bool` and `IsQuiet() bool` getter methods for use by commands
    - Import `github.com/charmbracelet/x/term` and `os`

    **styles.go** — Shared style definitions:
    - `borderStyle`: foreground color "240" (subtle gray, matches RoundedBorder aesthetic)
    - `headerStyle`: Bold(true), Foreground(Color("99")), Padding(0, 1)
    - `cellStyle`: Padding(0, 1)
    - `func statusStyle(status string) lipgloss.Style`: switch on strings.ToLower — "active"/"enabled" → Color("42") green, "inactive"/"disabled"/"deleted" → Color("196") red, "pending"/"draft" → Color("214") yellow, default plain
    - Import `charm.land/lipgloss/v2` and `strings`

    **table.go** — Table rendering:
    - `type TableDef struct` with Headers []string and StatusColumn int (-1 if none)
    - `func (f *Formatter) RenderTable(def TableDef, rows [][]string) error` — if quiet && !jsonMode return nil; create `table.New()` with `Border(lipgloss.RoundedBorder())`, `BorderStyle(borderStyle)`, `Headers(def.Headers...)`, `Rows(rows...)`, `StyleFunc` using headerStyle/cellStyle/statusStyle based on row/col, set `Width(f.width)` if width > 0; render via `fmt.Fprintln(f.writer, t.String())` (NOT lipgloss.Println — we need to write to f.writer, not always stdout)
    - `func Truncate(s string, maxLen int) string` — use `[]rune(s)` for Unicode-safe truncation, append "..." (3 dots, not ellipsis character) if truncated, return `string(runes[:maxLen-3]) + "..."` when len(runes) > maxLen
    - Import `charm.land/lipgloss/v2`, `charm.land/lipgloss/v2/table`, `fmt`

    Write tests FIRST (RED), then implement (GREEN). Tests should use a bytes.Buffer as the writer to capture output for assertions. For Formatter tests, construct with exported fields or use a test helper that sets writer to a buffer.
