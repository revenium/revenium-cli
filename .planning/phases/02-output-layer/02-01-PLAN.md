  <action>
    Create `internal/output/` package with three production files:

    **output.go** -- Formatter type:
    - `type Formatter struct` with fields: writer (io.Writer), errWriter (io.Writer), jsonMode (bool), quiet (bool), width (int), isTTY (bool)
    - `func New(jsonMode, quiet bool) *Formatter` -- detects TTY via `term.IsTerminal(os.Stdout.Fd())`, gets terminal width via `term.GetSize(os.Stdout.Fd())` (default 80 on failure), sets writer to `io.Discard` if quiet && !jsonMode
    - CRITICAL for FNDN-10: Use `colorprofile.Detect(os.Stdout, os.Environ())` to detect the color profile. Wrap `f.writer` with `colorprofile.NewWriter(os.Stdout, profile)` so that all output written through the Formatter automatically has ANSI codes downsampled or stripped based on the detected profile. This ensures NO_COLOR, TERM=dumb, and non-TTY outputs are handled correctly at write time without relying on `lipgloss.Println()`.
    - Expose `IsJSON() bool` and `IsQuiet() bool` getter methods for use by commands
    - Also provide `NewWithWriter(w io.Writer, errW io.Writer, jsonMode, quiet bool) *Formatter` for testing -- accepts explicit writers and skips TTY detection/colorprofile wrapping, so tests can use bytes.Buffer
    - Import `github.com/charmbracelet/colorprofile`, `github.com/charmbracelet/x/term`, and `os`

    **styles.go** -- Shared style definitions:
    - `borderStyle`: foreground color "240" (subtle gray, matches RoundedBorder aesthetic)
    - `headerStyle`: Bold(true), Foreground(Color("99")), Padding(0, 1)
    - `cellStyle`: Padding(0, 1)
    - `func statusStyle(status string) lipgloss.Style`: switch on strings.ToLower -- "active"/"enabled" -> Color("42") green, "inactive"/"disabled"/"deleted" -> Color("196") red, "pending"/"draft" -> Color("214") yellow, default plain
    - Import `charm.land/lipgloss/v2` and `strings`

    **table.go** -- Table rendering:
    - `type TableDef struct` with Headers []string and StatusColumn int (-1 if none)
    - `func (f *Formatter) RenderTable(def TableDef, rows [][]string) error` -- if quiet && !jsonMode return nil; create `table.New()` with `Border(lipgloss.RoundedBorder())`, `BorderStyle(borderStyle)`, `Headers(def.Headers...)`, `Rows(rows...)`, `StyleFunc` using headerStyle/cellStyle/statusStyle based on row/col, set `Width(f.width)` if width > 0; render via `fmt.Fprintln(f.writer, t.String())`. Because f.writer is wrapped with colorprofile.NewWriter(), ANSI codes are automatically stripped/downsampled for non-TTY and NO_COLOR environments -- no need to use lipgloss.Println().
    - `func Truncate(s string, maxLen int) string` -- use `[]rune(s)` for Unicode-safe truncation, append Unicode ellipsis "…" (U+2026, single character) if truncated, return `string(runes[:maxLen-1]) + "\u2026"` when len(runes) > maxLen. Note: maxLen-1 (not maxLen-3) because the ellipsis is a single rune.
    - Import `charm.land/lipgloss/v2`, `charm.land/lipgloss/v2/table`, `fmt`

    Write tests FIRST (RED), then implement (GREEN). Tests should use a bytes.Buffer as the writer to capture output for assertions. For Formatter tests, construct with NewWithWriter or exported fields to set writer to a buffer. For TestNoColor and TestNonTTY, test the colorprofile integration: create a Formatter using New() with NO_COLOR=1 set via t.Setenv, render a styled table, and assert output contains no ESC (\x1b) sequences.
