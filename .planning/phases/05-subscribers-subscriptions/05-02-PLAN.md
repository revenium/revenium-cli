**create.go** — `newCreateCmd()`:
- POST `/v2/api/subscriptions`
- Flags: `--description` (string, optional), `--subscriber-id` (string, optional), `--product-id` (string, optional)
- No required flags (field requirements uncertain from API docs — let API validate)
- Use `Flags().Changed()` for all fields to build body
- Error if no flags specified: "no fields specified"
- Renders result with renderSubscription

**update.go** — THE UNIQUE COMMAND for this phase:
- Args: ExactArgs(1) for subscription ID
- Flags:
  - `--patch` (bool, false) — "Use partial update (PATCH) instead of full update (PUT)"
  - `--description` (string) — "Subscription description"
  - `--subscriber-id` (string) — maps to body key "subscriberId"
  - `--product-id` (string) — maps to body key "productId"
- Logic:
  1. Build body using `Flags().Changed()` for each field flag
  2. Error if `len(body) == 0`: "no fields specified to update"
  3. Set `method := "PUT"`; if `patch` flag is true, set `method = "PATCH"`
  4. Call `cmd.APIClient.Do(c.Context(), method, "/v2/api/subscriptions/"+id, body, &result)`
  5. Render with renderSubscription
- Examples: full update, partial update with --patch

**delete.go** — `newDeleteCmd()`:
- DELETE `/v2/api/subscriptions/{id}`
- Uses `resource.ConfirmDelete("subscription", id, yes, cmd.Output.IsJSON())`
- Success message: "Deleted subscription %s.\n"
- Quiet mode check: `if !cmd.Output.IsQuiet()`
- Clone sources/delete.go exactly, changing resource name and path
