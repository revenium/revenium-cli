Clone the cmd/products/ package structure for teams CRUD, then add prompt-capture subcommands following cmd/models/pricing.go pattern. Specific details:

**teams.go** -- Parent command + helpers:
- Package: `teams`
- Cmd Use: "teams", Short: "Manage teams"
- Example: list, get, create examples using "revenium teams"
- tableDef: Headers `["ID", "Name"]`, StatusColumn: -1 (teams have no status field)
- toRows: extract "id" and "label" (use "label" for Name display -- it's the read-only display name auto-set to team name; fall back to "name" if "label" is empty)
- str(): identical helper (each package has its own per convention)
- renderTeam(): single-row render using same 2 columns
- init(): AddCommand for list, get, create, update, delete, plus `Cmd.AddCommand(promptCaptureCmd)` and `initPromptCapture()` -- exactly like models.go does with pricingCmd and initPricing()

**list.go** -- newListCmd():
- API: GET /v2/api/teams
- Empty: "No teams found." in text, empty array in JSON
- 2 examples: basic list, list with --json

**get.go** -- newGetCmd():
- Args: cobra.ExactArgs(1)
- API: GET /v2/api/teams/{id}
- Renders single team via renderTeam()
- 2 examples: basic get, get with --json

**create.go** -- newCreateCmd():
- Flags: --name (required string), --description (optional string)
- Body: always include "name"; include "description" only if Flags().Changed("description")
- API: POST /v2/api/teams
- Renders result via renderTeam()
- Mark --name required via MarkFlagRequired
- 2 examples: minimal create, create with description

**update.go** -- newUpdateCmd():
- Args: cobra.ExactArgs(1)
- Flags: --name (optional), --description (optional)
- Body: only include changed flags via Flags().Changed()
- Error if no fields specified: "no fields specified to update"
- API: PUT /v2/api/teams/{id}
- Renders result via renderTeam()
- 2 examples: update name, update multiple fields

**delete.go** -- newDeleteCmd():
- Clone from cmd/products/delete.go
- Change: "product" -> "team", API path -> "/v2/api/teams/"+id
- Uses resource.ConfirmDelete("team", id, yes, cmd.Output.IsJSON())
- 2 examples: delete with confirmation, delete with --yes

**prompt_capture.go** -- Parent subcommand + helpers (follows pricing.go pattern):
- var promptCaptureCmd with Use: "prompt-capture", Short: "Manage prompt capture settings for a team"
- Example: get and set examples
- initPromptCapture(): adds get and set subcommands to promptCaptureCmd
- promptCaptureTableDef: Headers `["Setting", "Value"]`, StatusColumn: -1
- renderPromptSettings(settings map[string]interface{}) error: iterates over settings map, skips "_links" key, creates rows of [key, fmt.Sprint(val)], sorts rows by key (import "sort"), renders via cmd.Output.Render(promptCaptureTableDef, rows, settings)

**prompt_capture_get.go** -- newPromptCaptureGetCmd():
- Use: "get <team-id>", Args: cobra.ExactArgs(1)
- Short: "View prompt capture settings for a team"
- API: GET /v2/api/teams/{args[0]}/settings/prompts into map[string]interface{}
- Renders via renderPromptSettings()
- 2 examples: basic get, get with --json

**prompt_capture_set.go** -- newPromptCaptureSetCmd():
- Use: "set <team-id>", Args: cobra.ExactArgs(1)
- Short: "Update prompt capture settings for a team"
- Flags: --enabled (optional bool), --max-prompt-length (optional int) -- these are best-guess fields; the map[string]interface{} approach handles whatever the API accepts
- Body: only include changed flags -- if Flags().Changed("enabled") add "enabled" to body; if Flags().Changed("max-prompt-length") add "maxPromptLength" to body
- Error if no fields specified: "no fields specified to update"
- API: PUT /v2/api/teams/{args[0]}/settings/prompts
- Renders result via renderPromptSettings()
- 2 examples: enable prompt capture, set max length
