Create 4 subcommand files:

**traces.go** -- `newTracesCmd()`:
- Endpoint: `/v2/api/traces`
- Use: `"traces"`, Short: `"Query AI traces"`
- This is the complex one: API returns individual trace entries, CLI must group by traceId.
- Implement `groupByTraceId(metrics []map[string]interface{}) []map[string]interface{}` that:
  1. Groups entries by `str(m, "traceId")`
  2. For each group: count entries, sum totalTokens (floatVal), sum totalCost (floatVal), keep first entry's model/source
  3. Preserves insertion order using an `order []string` slice
  4. Returns aggregated rows
- Table headers: `["Trace ID", "Entries", "Model", "Total Tokens", "Total Cost"]`
- In RunE: after API call, call `groupByTraceId(metrics)` before rendering table
- For JSON mode: pass raw API data (ungrouped) per user decision
- Empty message: "No traces found."

**squads.go** -- `newSquadsCmd()`:
- Endpoint: `/v2/api/squads` (execution metrics, not entities)
- Use: `"squads"`, Short: `"Query squad metrics"`
- Table headers: `["ID", "Name", "Executions", "Status"]` (adjust based on API response shape)
- Standard list pattern, same as AI metric subcommands
- Empty message: "No squad metrics found."

- Empty message: "No metrics found."
