  <files>cmd/metrics/metrics.go, cmd/metrics/metrics_test.go</files>
  <action>
Create `cmd/metrics/` directory and `metrics.go` with:

1. **Package declaration and imports:** `package metrics` with cobra, cmd, fmt, strings, time, net/url, encoding/json, internal/output imports.

2. **Package-level flag vars:** `var fromFlag string` and `var toFlag string`.

3. **Exported Cmd:** Parent `cobra.Command` with `Use: "metrics"`, `Short: "Query metrics and analytics"`, and 2-3 help examples showing `revenium metrics ai` and `revenium metrics completions --from ... --to ...`.

4. **init() function:** Register `--from` and `--to` as `PersistentFlags().StringVar()` on `Cmd`. Add all subcommands: `newAICmd()`, `newCompletionsCmd()`, `newAudioCmd()`, `newImageCmd()`, `newVideoCmd()`. Leave placeholder comments for traces, squads, api, tool-events (added in Plan 02).

5. **buildPath(base string) string helper:** Constructs API path with time range query parameters. When both `--from` and `--to` are empty, default to last 24 hours using `time.Now().UTC()` and `.Add(-24 * time.Hour)`. Format with `time.RFC3339`. Append as `startDate=` and `endDate=` query parameters using `url.QueryEscape()`. Handle `?` vs `&` separator correctly.

6. **formatNumber(n float64) string helper:** Format integer portion with comma grouping (e.g., 1234567 -> "1,234,567"). Handle negative numbers. Use `fmt.Sprintf("%.0f", n)` for the integer part, then insert commas every 3 digits from the right. Based on the `formatCurrency()` pattern in `cmd/alerts/budget.go` but without currency symbol or decimal places.

7. **str(m map[string]interface{}, key string) string helper:** Same pattern as every other resource package -- safely extract string value, return "" for missing/nil.

8. **floatVal(m map[string]interface{}, key string) float64 helper:** Same as `cmd/alerts/budget.go` -- handle float64 and json.Number types.

- `TestBuildPath_Defaults` -- verifies 24h default when no flags set (check startDate/endDate params present)
- `TestBuildPath_WithFlags` -- verifies flag values are passed through

- `TestFormatNumber` -- test cases: 0->"0", 999->"999", 1000->"1,000", 1234567->"1,234,567", -1234->"-1,234"
