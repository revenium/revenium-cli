# Phase 11: Distribution & Shell Completions - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

GoReleaser configuration for cross-platform binary releases, Homebrew tap formula for macOS/Linux install, and shell completion scripts for bash/zsh/fish. This is the final phase — packaging and distribution of the completed CLI.

</domain>

<decisions>
## Implementation Decisions

### GoReleaser & Release Scope
- Claude decides all GoReleaser configuration details (target platforms, archive format, changelog)
- Standard GoReleaser setup for Go CLI projects

### Homebrew Tap Setup
- Tap repository: `revenium/homebrew-tap` — installs via `brew install revenium/tap/revenium`
- GitHub org: `revenium` — both CLI repo and tap repo under `github.com/revenium/`
- User will create the tap repo manually — plan assumes it exists
- GoReleaser auto-publishes the formula to the tap repo on each release (brews section in .goreleaser.yml)
- Shell completions bundled into Homebrew formula — install automatically with `brew install`

### Shell Completion Strategy
- Scripts only via `revenium completion bash|zsh|fish` — standard Cobra behavior, no install helpers
- Help text includes usage examples showing how to source the script for each shell
- Cobra auto-generated descriptions — no custom completion descriptions
- Completions generated during GoReleaser build and bundled into Homebrew formula for automatic install

### Claude's Discretion
- GoReleaser target platforms (linux/mac/windows, amd64/arm64)
- Archive format (tar.gz vs zip)
- Changelog generation approach
- Exact Homebrew formula structure
- Completion help text wording and examples
- Whether to add a completion test

</decisions>

<specifics>
## Specific Ideas

- Shell completions should "just work" for Homebrew users — no manual setup needed
- The `revenium completion` help should be immediately actionable for non-Homebrew users
- GoReleaser should embed the same ldflags already in the Makefile (`internal/build.Version`, etc.)

</specifics>
