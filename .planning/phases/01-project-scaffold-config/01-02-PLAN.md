    - TestErrorMapping404: 404 response returns APIError with "Resource not found." message
    - TestErrorMapping500: 5xx response returns APIError with "Revenium API error" message

    - TestVerboseLogging: When Verbose=true, request method+URL and response status are written to stderr

Create `internal/api/client.go`:
- Package `api`
- Import `internal/build` for User-Agent, import `internal/errors` for APIError
- Type `Client struct { BaseURL string; APIKey string; HTTPClient *http.Client; Verbose bool }`
- `func NewClient(baseURL, apiKey string, verbose bool) *Client`: creates Client with `&http.Client{Timeout: 30 * time.Second}`
- `func (c *Client) Do(ctx context.Context, method, path string, body, result interface{}) error`:
  - Marshal body to JSON if non-nil, create request with `http.NewRequestWithContext`
  - Set headers: `x-api-key`, `Content-Type: application/json`, `Accept: application/json`, `User-Agent: revenium-cli/{build.Version}`
  - If Verbose: log `> METHOD URL` and `< STATUS` to stderr
  - Verbose must mask API key: show `x-api-key: ****{last4}` (per research pitfall 7)
  - Execute request, handle connection errors with "Could not connect to api.revenium.ai. Check your network connection."
  - `defer` body drain (`io.Copy(io.Discard, resp.Body)`) and close
  - If status >= 400: call `mapHTTPError(resp)` and return error
  - If result non-nil: decode JSON into result
- `func mapHTTPError(resp *http.Response) error`:
  - Read body bytes
  - 401: "Invalid API key. Run `revenium config set key <your-key>` to fix."
  - 403: "Access denied. Your API key may not have permission for this operation."
  - 404: "Resource not found."
  - 5xx: "Revenium API error. Try again later or contact support."
  - Other 4xx: "Request failed (HTTP {code})."
  - Return `&errors.APIError{StatusCode, Message, Body}`

Tests use `net/http/httptest.NewServer` to mock API responses.
