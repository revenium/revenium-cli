<task type="auto" tdd="true">
  <name>Task 2: Config package with load, set, and env var override</name>
  <files>internal/config/config.go, internal/config/config_test.go</files>
  <behavior>
    - TestLoadConfig: Loading config from a temp YAML file returns correct APIKey and APIURL values
    - TestLoadConfigMissing: Loading when no config file exists returns empty APIKey and default APIURL
    - TestLoadConfigDefault: APIURL defaults to "https://api.revenium.ai/profitstream" when not set
    - TestSetConfig: Setting "api-key" writes value to YAML file; setting "api-url" writes value to YAML file
    - TestSetConfigCreatesDir: Set creates the config directory if it does not exist
    - TestEnvOverrideAPIKey: Setting REVENIUM_API_KEY env var overrides config file api-key value
    - TestEnvOverrideAPIURL: Setting REVENIUM_API_URL env var overrides config file api-url value
  </behavior>
  <action>
Create `internal/config/config.go`:
- Package `config`
- Type `Config struct { APIKey string; APIURL string }`
- `func configDir() (string, error)`: uses `os.UserHomeDir()` + `/.config/revenium` (NOT os.UserConfigDir -- returns wrong path on macOS per user decision)
- `func configPath() (string, error)`: configDir + `/config.yaml`
- `func Load() (*Config, error)`:
  - Construct path via configDir()
  - `viper.SetConfigName("config")`, `viper.SetConfigType("yaml")`, `viper.AddConfigPath(dir)`
  - `viper.SetEnvPrefix("REVENIUM")`, `viper.AutomaticEnv()`
  - `viper.SetEnvKeyReplacer(strings.NewReplacer("-", "_"))` for REVENIUM_API_KEY mapping
  - Bind env: `viper.BindEnv("api-key")`, `viper.BindEnv("api-url")`
  - `viper.SetDefault("api-url", "https://api.revenium.ai/profitstream")`
  - ReadInConfig -- ignore ConfigFileNotFoundError, return other errors
  - Return Config with `APIKey: viper.GetString("api-key")`, `APIURL: viper.GetString("api-url")`
- `func Set(key, value string) error`:
  - Get configDir, `os.MkdirAll(dir, 0o700)`
  - Get configPath, `viper.SetConfigFile(path)`
  - `viper.ReadInConfig()` (ignore not-found to preserve existing values)
  - `viper.Set(key, value)`, `viper.WriteConfigAs(path)`

IMPORTANT: Tests must use temp directories to avoid touching real user config. Each test should:
- Create a temp dir, set config path override OR use a test helper that overrides configDir
- Add a package-level `var configDirOverride string` that configDir() checks first (for testing)
- Clean up after tests

Use `viper.Reset()` between tests to avoid state leakage (Viper uses global state).
  </action>
